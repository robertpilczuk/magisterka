markdown# 11. Internacjonalizacja (i18n) — przełącznik języka PL/EN

## Co zostało zbudowane

Kompletny system tłumaczeń dla interfejsu użytkownika obsługujący dwa języki:
polski (domyślny) i angielski. Przełączanie języka odbywa się bez przeładowania
strony — wszystkie komponenty aktualizują się natychmiast po kliknięciu przycisku
w prawym górnym rogu.

---

## Dlaczego poprzednie rozwiązanie nie działało

Pierwsza wersja przełącznika używała `window.location.reload()` — czyli po prostu
przeładowywała całą stronę. Miało to dwie wady:

1. **Resetowanie stanu** — jeśli użytkownik wyszukał dane użytkownika i przełączył
   język, wyniki znikały i trzeba było szukać od nowa.
2. **Brak reaktywności** — React nie wiedział że język się zmienił, bo zmienna
   `currentLang` w `i18n.js` to zwykła zmienna JavaScript, a nie stan Reacta.

---

## Architektura rozwiązania

### Czym jest React Context?

React Context to mechanizm który pozwala przekazać dane do wszystkich komponentów
w aplikacji bez konieczności ręcznego przekazywania ich przez każdy poziom drzewa
komponentów (tzw. "prop drilling").

Wyobraź sobie budynek z centralnym systemem ogrzewania — zamiast każdy pokój
ogrzewać osobno, jedna centrala dostarcza ciepło wszędzie. Context działa
podobnie: jeden Provider dostarcza dane (język, funkcję zmiany języka, funkcję
tłumaczenia) do wszystkich komponentów które o to poproszą.

### Schemat działania
main.jsx
└── <LangProvider>          ← "centrala" — trzyma aktywny język
└── <App>
├── <SearchBar>     ← useLang() → pobiera t(), lang
├── <UserProfile>   ← useLang() → pobiera t(), lang
├── <GenreFilter>   ← useLang() → pobiera t(), lang
└── ...             ← każdy komponent subskrybuje kontekst

Kiedy użytkownik klika przełącznik języka:
1. `setLang('EN')` w `LangProvider` aktualizuje stan `lang`
2. React automatycznie rerenderuje **wszystkie** komponenty które używają `useLang()`
3. Każde wywołanie `t('klucz')` zwraca teraz tekst w nowym języku

## Pliki i ich role

### `frontend/src/LangContext.jsx` — serce systemu

```jsx
export function LangProvider({ children }) {
  const [lang, setLangState] = useState('PL')

  function setLang(newLang) {
    setLangModule(newLang)   // aktualizuje zmienną w i18n.js
    setLangState(newLang)    // aktualizuje stan Reacta → wyzwala rerenderowanie
  }

  function t(key, subkey = null) {
    // czyta z translations[lang] — gdy lang się zmienia, t() zwraca nowe wartości
    const langData = translations[lang]
    // ...
  }

  return (
    <LangContext.Provider value={{ lang, setLang, t }}>
      {children}
    </LangContext.Provider>
  )
}

export function useLang() {
  return useContext(LangContext)  // hook do pobierania kontekstu w komponentach
}
```

**Kluczowy szczegół:** funkcja `t()` jest zdefiniowana wewnątrz `LangProvider`
i czyta `lang` ze stanu Reacta. Dzięki temu gdy `lang` się zmienia, React wie
że wszystkie komponenty używające `t()` muszą się przerenderować.

### `frontend/src/i18n.js` — słownik tłumaczeń

Plik zawiera dwa obiekty: `translations.PL` i `translations.EN`, każdy z tymi
samymi kluczami. Struktura jest hierarchiczna — klucze są pogrupowane tematycznie:

| Sekcja | Zawartość |
|--------|-----------|
| `nav` | Tytuł aplikacji, nazwy zakładek |
| `search` | Wyszukiwarka użytkownika |
| `similar` | Filtr podobnych użytkowników |
| `gender` | Etykiety płci |
| `age` | Przedziały wiekowe |
| `occupation` | 21 zawodów (0–20) |
| `profile` | Nagłówki profilu użytkownika |
| `viewer_type` | Typy widzów (Krytyk, Sceptyk...) |
| `viewer_desc` | Opisy typów widzów |
| `taste` | Profil filmowy (Lubi/Neutralne/Nie lubi) |
| `recommendations` | Panele rekomendacji |
| `comparison` | Porównanie użytkowników |
| `validation` | Walidacja modelu |
| `genre_filter` | Filtr gatunków |
| `genres` | Nazwy 18 gatunków filmowych |
| `new_user` | Zakładka "Moje rekomendacje" |
| `deep` | Zakładka "Pogłębiona analiza" |

Użycie funkcji `t()`:
```js
t('nav.title')           // → "Film Recommender"
t('occupation', 4)       // → "Student" (PL) lub "College / grad student" (EN)
t('genres', 'Action')    // → "Akcja" (PL) lub "Action" (EN)
```

### `frontend/src/utils.js` — style typów widzów

Funkcja `getRatingStyle()` została rozbudowana o obsługę tłumaczeń. Przyjmuje
teraz drugi argument `t` — funkcję tłumaczącą:

```js
// przed:
getRatingStyle(avg)
// zwracała hardcoded: { text: 'Krytyk', desc: '...', color: '#c0392b' }

// po:
getRatingStyle(avg, t)
// zwraca: { text: t('viewer_type', 'critic'), desc: t('viewer_desc', 'critic'), color: '#c0392b' }
```

Mapowanie średniej oceny na typ widza:

| Wynik (0–10) | Klucz | PL | EN |
|---|---|---|---|
| 0–2 | `critic` | Krytyk | Critic |
| 3–4 | `skeptic` | Sceptyk | Skeptic |
| 5 | `picky` | Wybredny widz | Picky viewer |
| 6 | `typical` | Typowy widz | Typical viewer |
| 7 | `friendly` | Życzliwy widz | Friendly viewer |
| 8 | `optimist` | Optymista | Optimist |
| 9 | `cinephile` | Miłośnik kina | Cinephile |
| 10 | `enthusiast` | Entuzjasta | Enthusiast |

## Komponenty które zostały zaktualizowane

Wszystkie 10 komponentów frontendu oraz `App.jsx` zostały zaktualizowane.
Każdy używa teraz wzorca:

```jsx
import { useLang } from '../LangContext'

export default function MojKomponent() {
  const { t } = useLang()
  // ...
  return <div>{t('klucz.podklucz')}</div>
}
```

Lista zaktualizowanych plików:
- `App.jsx` — zakładki, nagłówek, sekcja porównania
- `SearchBar.jsx` — placeholder, przycisk, komunikat błędu
- `UserProfile.jsx` — etykiety pól, płeć, wiek, zawód, typ widza
- `RecommendationCard.jsx` — opis wyniku logistycznego
- `ValidationChart.jsx` — nagłówki, metryki, kolumny tabeli
- `SimilarUsersFilter.jsx` — filtry, wyniki, karty użytkowników
- `UserComparison.jsx` — nagłówki, etykiety demograficzne
- `UserTasteProfile.jsx` — sekcje, gatunki, tooltopy, paginacja
- `GenreFilter.jsx` — przyciski gatunków, licznik ukrytych
- `NewUserFlow.jsx` — wszystkie 3 kroki, przełącznik paneli
- `DeepAnalysisFlow.jsx` — wszystkie 4 kroki, kryteria, suwaki

## Jak dodać nowe tłumaczenie

1. Dodaj klucz do obu sekcji (`PL` i `EN`) w `i18n.js`:
```js
PL: {
  moja_sekcja: {
    nowy_klucz: 'Tekst po polsku'
  }
}
EN: {
  moja_sekcja: {
    nowy_klucz: 'Text in English'
  }
}
```

2. Użyj w komponencie:
```jsx
const { t } = useLang()
// ...
<div>{t('moja_sekcja.nowy_klucz')}</div>
```
